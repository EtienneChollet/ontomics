# ontomics v0.3.0

## L4 Behavioral Embedding Refactor

Replaces the pseudocode generation pipeline with raw function body text embedding
via CodeRankEmbed. Functions are now clustered by behavioral similarity using their
actual source code, eliminating ~1,200 lines of per-language pseudocode generators
and their associated bugs.

### What changed

- **New behavioral embedding model**: CodeRankEmbed (`nomic-ai/CodeRankEmbed`,
  137M params, 768-dim) embeds raw function bodies for behavioral clustering.
  Concept embeddings remain on BGE-small.
- **Pluggable embedding backend**: `EmbeddingModel` trait supports multiple
  backends (BGE-small, CodeRankEmbed, Jina Code, GTE-ModernBERT).
- **Deleted `pseudocode.rs`**: Removed `PseudocodeStep` enum and all four
  per-language generators (~1,200 lines).
- **FunctionBody improvements**: Added `language` and `was_truncated` fields,
  plus `dedent()` indentation normalization for consistent embedding regardless
  of nesting depth.
- **Cache auto-invalidation**: Existing v0.2.x caches will re-index automatically
  on first run. No manual action required.

### Net change: -1,676 lines (429 added, 2,105 deleted, 14 files)

### Performance

Measured on voxelmorph (13 files) and interseg3d (163 files) with CUDA, release
builds:

- **Cold cache wall time**: +17-37% (second model loading overhead; amortizes
  with repo size — larger repos see smaller regression)
- **Warm cache (MCP server)**: unchanged in both wall time and memory
- **Cold cache memory**: +29% during indexing (two models loaded concurrently)
- **Cache size**: +13-18% (768-dim vectors replacing 384-dim)

### Behavioral cluster quality

Inspected 21 clusters on voxelmorph. All 10 non-singleton clusters are
thematically coherent: distance transforms cluster together, data generators
cluster together, spatial transform operations cluster together, file I/O pairs
cluster together. No embedding collapse observed.

### Breaking changes

- Existing `.ontomics/index.db` caches will be invalidated and rebuilt on first
  run. This is a one-time cost.
- `[logic]` config section no longer accepts `max_pseudocode_lines` or
  `min_pseudocode_steps` (removed — these were pseudocode-specific).

### MCP tool surface

No changes to tool signatures or response schemas. All existing tools continue
to work as before.
